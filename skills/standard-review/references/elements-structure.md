# 标准文件的要素和结构规则

本文档基于 GB/T 1.1-2020 第6-8章的规定。

## 文件层次结构

标准文件可能具有的层次（从大到小）：

| 层次 | 编号示例 | 说明 |
|------|---------|------|
| **部分** | XXXX.1, XXXX.2 | 文件的第一层次划分 |
| **章** | 5 | 层次划分的基本单元 |
| **条** | 5.1, 5.1.1, 5.1.1.1 | 章内有编号的细分，最多5层 |
| **段** | [无编号] | 章或条内没有编号的细分 |
| **列项** | ——、a)、1) | 并列项目的列举 |

## 文件要素分类

### 按作用分类
- **规范性要素**：构成文件主体规定的部分
- **资料性要素**：提供附加信息和说明的部分

### 按状态分类
- **必备要素**：文件必须包含的要素
- **可选要素**：根据需要选择的要素

## 文件要素构成表

| 要素 | 类别 | 构成 | 允许的表述形式 |
|------|------|------|--------------|
| **封面** | 必备/资料性 | 附加信息 | 标明文件信息 |
| **目次** | 可选/资料性 | 附加信息 | 列表（自动生成） |
| **前言** | 必备/资料性 | 附加信息 | 条文、注、脚注、指明附录 |
| **引言** | 可选/资料性 | 附加信息 | 条文、图、表、数学公式、注、脚注、指明附录 |
| **范围** | 必备/规范性 | 条款+附加信息 | 条文、表、注、脚注 |
| **规范性引用文件** | 必备/资料性 | 附加信息 | 清单、注、脚注 |
| **术语和定义** | 必备/规范性 | 条款+附加信息 | 条文、图、数学公式、示例、注、引用、提示 |
| **符号和缩略语** | 可选/规范性 | 条款+附加信息 | 清单、注、脚注 |
| **分类和编码/系统构成** | 可选/规范性 | 条款+附加信息 | 条文、图、表、注、脚注、引用、提示、指明附录 |
| **总体原则和/或总体要求** | 可选/规范性 | 条款+附加信息 | 条文、图、表、数学公式、示例、注、脚注、引用、提示、指明附录 |
| **核心技术要素** | 必备/规范性 | 条款+附加信息 | 条文、图、表、数学公式、示例、注、脚注、引用、提示、指明附录 |
| **其他技术要素** | 可选/规范性 | 条款+附加信息 | 条文、图、表、数学公式、示例、注、脚注、引用、提示、指明附录 |
| **参考文献** | 可选/资料性 | 附加信息 | 清单、脚注 |
| **索引** | 可选/资料性 | 附加信息 | 列表（自动生成） |

**注**：规范性引用文件、术语和定义的章节号和标题设置是必备的，内容的有无根据具体情况进行选择。

## 核心技术要素（不同功能类型标准）

| 标准功能类型 | 核心技术要素 | 条款类型 |
|------------|------------|---------|
| **术语标准** | 术语和定义 | 陈述 |
| **符号标准** | 符号和缩略语 | 陈述 |
| **分类标准** | 分类和编码 | 陈述 |
| **试验标准** | 试验方法 | 指示为主 |
| **规范标准** | 技术要求 | 要求为主 |
| **规程标准** | 操作程序/工作规程 | 指示为主 |
| **指南标准** | 指导和建议 | 推荐为主 |

## 重要要素编写规则

### 1. 文件名称

**组成元素**（顺序从一般到特殊）：
- 引导元素（可选）：表示文件所属领域
- 主体元素（必备）：表示标准化对象
- 补充元素（可选）：表示对象的特殊方面或区分信息

**示例**：
- ✅ 农业机械和设备 散装物料机械 技术规范（有引导元素）
- ✅ 工业用过硼酸钠 堆积密度测定（无引导元素）
- ✅ 咖啡研磨机（省略补充元素）

**名称中的词语**：
- 应含有表示标准功能类型的词语（术语、符号、分类、编码、试验方法、规范、规程、指南等）
- 不应包含"······标准""······国家标准"等词语

### 2. 范围

**内容要求**：
- 简明陈述文件涵盖的主题及其适用范围
- 必要时说明文件不适用的范围
- 使用**陈述句**，不使用要求型或推荐型条款

**示例**：
- ✅ "本文件规定了×××的术语和定义"
- ✅ "本文件适用于×××"
- ❌ "本文件应适用于×××"（错误）

### 3. 规范性引用文件

**编写规则**：
- 引导语："下列文件中的内容通过文中的规范性引用而构成本文件必不可少的条款。其中，注日期的引用文件，仅该日期对应的版本适用于本文件；不注日期的引用文件，其最新版本（包括所有的修改单）适用于本文件。"
- 按文件编号顺序排列
- 无规范性引用文件时，应说明："本文件没有规范性引用文件。"

### 4. 术语和定义

**术语条目构成**（按顺序）：
1. 条目编号（必备）
2. 术语（必备）
3. 英文对应词（可选）
4. 定义（必备）
5. 示例（可选）
6. 注（可选）

**编写要点**：
- 定义应准确、简明、自洽
- 避免循环定义
- 使用陈述句
- 必要时给出来源

**示例**：
```
3.1.1
标准 standard
为了在一定范围内获得最佳秩序，经协商一致制定并由公认机构批准，共同使用和重复使用的一种规范性文件。
注：标准宜以科学、技术和经验的综合成果为基础，以促进最佳社会效益为目的。
```

### 5. 前言

**必备内容**：
- 本文件的起草规则声明
- 与之前版本的关系说明（如适用）
- 与国际文件的关系说明（如适用）
- 归口单位信息
- 起草单位和起草人

**不允许的条款类型**：
- 不应包含要求
- 不应包含推荐

### 6. 附录

**类型**：
- **规范性附录**：给出正文的补充或附加条款
- **资料性附录**：给出有助于理解或使用文件的附加信息

**编号规则**：
- 使用大写拉丁字母：附录A、附录B等
- 编号之下标明"(规范性)"或"(资料性)"
- 附录内的图、表、公式重新从1开始编号，前加字母

**示例**：
- 附录 A（规范性）层次编号示例
- 附录 B（资料性）参考文献示例

## 层次编号规则

### 部分编号
- 使用下脚点连接：XXXX.1、XXXX.2
- 部分名称包含"第 * 部分："

### 章编号
- 使用阿拉伯数字：1、2、3...
- 从"范围"开始连续编号

### 条编号
- 使用下脚点连接：5.1、5.1.1、5.1.1.1
- 最多分到第五层次
- 一个层次中有一个以上的条时才可设条

### 列项编号
- 第一层次：——（破折号）或 a)、b)、c)
- 第二层次：•（间隔号）或 1)、2)、3)

## 审校要点

### 结构完整性检查
1. ✅ 必备要素是否齐全（封面、前言、范围、规范性引用文件标题、术语和定义标题、核心技术要素）
2. ✅ 要素顺序是否正确
3. ✅ 层次划分是否合理（条的层次不超过5层）

### 编号规范性检查
1. ✅ 章编号是否连续
2. ✅ 条编号是否正确（使用下脚点）
3. ✅ 列项符号是否正确
4. ✅ 图、表、公式编号是否从1开始连续

### 内容规范性检查
1. ✅ 文件名称是否符合规则
2. ✅ 范围是否使用陈述句
3. ✅ 术语条目结构是否完整
4. ✅ 前言是否包含禁用的条款类型
5. ✅ 附录是否正确标明规范性或资料性
