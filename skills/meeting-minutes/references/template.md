# 会议纪要模板

## 标准格式

会议纪要应包含以下部分，按顺序组织：

### 1. 会议基本信息
```markdown
# [会议主题]

**会议时间**：YYYY-MM-DD HH:MM  
**会议地点**：[地点/线上平台]  
**记录人**：[姓名]  
```

### 2. 参会人员
```markdown
## 参会人员

- [姓名] - [职位/角色]
- [姓名] - [职位/角色]
- ...
```

### 3. 会议议程（可选）
```markdown
## 会议议程

1. [议题一]
2. [议题二]
3. ...
```

### 4. 讨论要点
```markdown
## 讨论要点

### [议题一标题]
- 讨论内容摘要
- 关键观点和意见
- 相关数据或事实

### [议题二标题]
- 讨论内容摘要
- 关键观点和意见
```

讨论要点应该：
- 按议题分组，使用三级标题
- 提取核心内容，去除冗余
- 保留关键数据和事实
- 记录重要的不同意见

### 5. 决策事项
```markdown
## 决策事项

1. **[决策主题]**：具体决策内容和理由
2. **[决策主题]**：具体决策内容和理由
```

决策事项应该：
- 清晰明确，可执行
- 包含决策依据（如有）
- 使用加粗突出决策主题

### 6. 行动项
```markdown
## 行动项

| 序号 | 任务描述 | 负责人 | 截止时间 | 状态 |
|------|---------|--------|---------|------|
| 1 | [具体任务] | [姓名] | YYYY-MM-DD | 待办 |
| 2 | [具体任务] | [姓名] | YYYY-MM-DD | 待办 |
```

行动项应该：
- 使用表格格式清晰展示
- 任务描述具体、可衡量
- 明确责任人（单一责任人）
- 设定明确的截止时间
- 初始状态统一为"待办"

### 7. 待办事项（可选）
```markdown
## 待办事项

- [ ] [待跟进事项]
- [ ] [待确认事项]
- [ ] [待讨论事项]
```

待办事项用于记录：
- 需要跟进但未分配责任人的事项
- 需要进一步确认的问题
- 下次会议需要讨论的议题

### 8. 备注（可选）
```markdown
## 备注

- [其他需要记录的信息]
- [相关文档链接]
```

## 格式要求

1. **使用 Markdown 格式**：标题、列表、表格、加粗等
2. **层次清晰**：一级标题为会议主题，二级标题为各部分，三级标题为子议题
3. **信息完整**：必须包含会议基本信息、参会人员、讨论要点、决策事项、行动项
4. **语言简洁**：提取关键信息，避免冗长描述
5. **时间格式统一**：使用 YYYY-MM-DD HH:MM 格式
6. **表格对齐**：行动项表格保持格式整齐

## 示例

```markdown
# 产品需求评审会议

**会议时间**：2024-01-15 14:00  
**会议地点**：会议室A / 腾讯会议  
**记录人**：张三  

## 参会人员

- 李四 - 产品经理
- 王五 - 技术负责人
- 张三 - 开发工程师
- 赵六 - UI设计师

## 讨论要点

### 用户登录功能优化

- 当前登录流程存在3步操作，用户反馈较为繁琐
- 技术团队建议采用手机号+验证码的快捷登录方式
- UI设计需要在1周内提供新的登录页面设计稿
- 预计开发周期2周，可在下个迭代完成

### 数据统计功能需求

- 产品提出需要增加用户行为数据统计面板
- 需要展示DAU、留存率、转化率等核心指标
- 技术团队确认需要接入数据分析平台，预计需要3周开发时间
- 暂定优先级为P1，排入下下个迭代

## 决策事项

1. **登录功能优化**：确认采用手机号+验证码方案，排入下个迭代（v2.1）
2. **数据统计功能**：确认需求，优先级P1，排入v2.2迭代
3. **设计评审**：UI设计稿完成后需进行一轮评审再进入开发

## 行动项

| 序号 | 任务描述 | 负责人 | 截止时间 | 状态 |
|------|---------|--------|---------|------|
| 1 | 完成登录页面设计稿 | 赵六 | 2024-01-22 | 待办 |
| 2 | 评审登录页面设计稿 | 李四、王五 | 2024-01-23 | 待办 |
| 3 | 输出数据统计功能详细PRD | 李四 | 2024-01-25 | 待办 |
| 4 | 评估数据分析平台接入方案 | 王五 | 2024-01-26 | 待办 |

## 待办事项

- [ ] 确认数据分析平台的采购预算
- [ ] 下次会议讨论数据统计功能的具体指标定义

## 备注

- 相关文档：[产品需求文档 v2.0](链接)
- 下次会议时间：2024-01-29 14:00
```
